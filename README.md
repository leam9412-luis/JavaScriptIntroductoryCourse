## Descripción del Curso
Este curso está diseñado para enseñarte las bases para el desarrollo web. Aprenderás no solo la sintaxis, sino también las mejores prácticas, patrones de diseño y resolución de problemas reales.

> **💡 Nota:** Este material se actualiza periódicamente con nuevos módulos y/o retos de código.

## Requisitos Previos
Antes de empezar, asegúrate de tener instalado:
* **Lenguaje/Entorno**: [Node.js](https://nodejs.org/).
* **Editor de Código (IDE)**: Recomiendo VS Code —[VS Code](https://code.visualstudio.com/)— pero puedes usar el IDE de tu preferencia.
* **Git**: Para gestionar las versiones y clonar el repo. [Git](https://git-scm.com/).
* **Github Desktop** (_Opcional_): Una interfaz gráfica para gestionar tus repos. Importante entender antes como funciona Git para poder usar [GitHub Desktop](https://github.com/apps/desktop).

## Contribuciones
Si quieres añadir un ejercicio, ¡los Pull Requests son bienvenidos! <br>

Sigue estos pasos:

1. A esta punto, ya deberias tener un fork del proyecto en tu local. Si no lo tienes aún, haz un Fork(Clonar) del proyecto (Sección instalación y configuración).
Una vez lo hayas hecho, crea una rama para trabajar tu ejercicio. **¡NOTA!: Asegurate de tener los últimos cambios del repo en tu repo local**. Para esto puede usar este comando para comprobar y hacer pull si es necesario (Asegurate de tener el area de trabajo limpia):
   ```bash
   git fetch

2. Crea una rama para trabajar. Te recomiendo esta estructura:  **_AJH-CreacionDelReadme-G32026_** donde:
  * **AJH** son tus iniciales.
  * **CreacionDelReadme** es una breve descripción del ejercicio.
  * **G32026** es el grupo al que perteneces y el año.
    Lo puedes hacer con este comando(Recuerda estar sobre la rama principal):
    ```bash
    git checkout -b AJH-CreacionDelReadme-G32026.
3. Navega a la carpeta **estudiantes/grupo-año**.
   ```bash
   cd estudiantes/grupo-año

4. Si es tu primera vez haciendo una contribución, crea una carpeta con tu nombre con la estructura **tu-nombre**.
   ```bash
   mkdir arturo-hoyos

5. Haz un commit de tus cambios.

6. Envía el Pull Request.
> **💡 Nota!:** Para efectos practicos del curso, los Pull Request se harán a la rama **main** directamente.
